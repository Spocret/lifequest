# Настройка ЮКассы для LifeQuest (позже)

Краткий чеклист: что сделать в личном кабинете ЮКассы и в приложении, чтобы оплата подписки Pro (490 ₽ / месяц) зачисляла пользователю **30 дней** доступа.

## 1. ЮКасса: регистрация и магазин

1. Зарегистрируйся в [ЮКасса](https://yookassa.ru/) и подключи приём платежей (ИП или ООО — по твоей схеме).
2. В личном кабинете создай **магазин** и выпиши:
   - **shopId** (идентификатор магазина)
   - **Секретный ключ** (для серверных запросов, не класть в `VITE_*`).

## 2. Сценарий оплаты

В Mini App кнопка «ЮКасса» ведёт на **`VITE_YOOKASSA_PAYMENT_URL`** — это должен быть **твой** URL, который:

- принимает пользователя (например, с `user_id` в query или одноразовым токеном),
- на **сервере** создаёт платёж через API ЮКассы (`POST /v3/payments` с `confirmation.type = redirect` и `return_url` обратно в приложение),
- редиректит пользователя на `confirmation.confirmation_url` от ЮКассы.

Важно: **секретный ключ ЮКассы** используется только на бэкенде (Vercel Serverless, отдельный API), не в браузере.

## 3. После успешной оплаты

ЮКасса шлёт **webhook** на твой endpoint (настрой URL в кабинете ЮКассы).

Обработчик должен:

1. Проверить подпись / данные уведомления по [документации ЮКассы](https://yookassa.ru/developers/using-api/webhooks).
2. Убедиться, что платёж **успешен** (`payment.status === 'succeeded'`).
3. Определить **пользователя приложения** (из `metadata` платежа ты заранее передаёшь `user_id` при создании платежа).
4. Вызвать продление Pro в базе — в проекте уже есть endpoint:

   - **`POST /api/subscription/extend-pro`**
   - Заголовок: `Authorization: Bearer <SUBSCRIPTION_WEBHOOK_SECRET>`
   - Тело JSON: `{ "userId": "<uuid из Supabase>", "months": 1 }`

   В Vercel задай переменную **`SUBSCRIPTION_WEBHOOK_SECRET`** (длинная случайная строка) и используй тот же секрет только на своём сервере, который дергает этот API после проверки платежа.

5. Альтернатива: тот же код продления выполнить **напрямую** в webhook-обработчике через **Supabase Service Role** (как в `api/subscription/extend-pro.ts`), без отдельного HTTP-вызова.

## 4. Переменные окружения (фронт / Vercel)

| Переменная | Назначение |
|------------|------------|
| `VITE_YOOKASSA_PAYMENT_URL` | Публичная ссылка на **твою** страницу оплаты (редирект из приложения). Пример: `https://твой-домен.vercel.app/pay` с логикой создания платежа. |
| `VITE_SUBSCRIPTION_MANAGE_URL` | (Опционально) Ссылка «Управление подпиской» в профиле — личный кабинет ЮКассы, бот или инструкция. |
| `SUBSCRIPTION_WEBHOOK_SECRET` | Секрет для `POST /api/subscription/extend-pro` (только сервер). |

Секреты ЮКассы и Supabase Service Role **не** добавляй в префикс `VITE_`.

## 5. База данных

Колонка **`users.pro_until`** задаёт дату окончания оплаченного Pro. Миграция: `supabase/migrations/20260404000000_users_pro_until.sql`.

Cron **`/api/cron/trial`** ежедневно снимает Pro, если `pro_until` в прошлом.

## 6. Проверка

1. Тестовый платёж в [тестовом магазине ЮКассы](https://yookassa.ru/developers/payment-acceptance/testing-and-going-live/testing) (тестовые карты в документации).
2. Убедиться, что в `users` у тестового пользователя обновились `plan = 'pro'` и `pro_until` ≈ +30 дней от момента продления.
3. В приложении: **Профиль** — отображаются оставшиеся дни и дата окончания.
